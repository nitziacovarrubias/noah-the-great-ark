# Genesis: The First Garden

**Developed by Diego Mejía & Nitzia Covarrubias**

A 2D pixel-art game inspired by the story of creation in Genesis.


## Project structure

```text
genesis-the-first-garden/       # Root folder of the Godot project
├── .godot/                     # Godot-generated cache, import data, and editor metadata
├── assets/                     # Raw game assets used by the project
│   ├── audio/                  # Music, ambience, and sound effects
│   ├── backgrounds/            # Large background images and scenery art
│   ├── fonts/                  # Fonts used in menus, titles, and UI
│   ├── icons/                  # Small interface icons and cursor graphics
│   ├── sprites/                # Character sprites, props, and animation sheets
│   ├── tilesets/               # Tile atlases and environment tiles
│   └── ui/                     # UI artwork such as panels, frames, and buttons
├── scenes/                     # Godot scenes that assemble the game
│   ├── core/                   # Base scenes like boot, loaders, or transitions
│   ├── menus/                  # Main menu, options, credits, and pause scenes
│   ├── levels/                 # Playable levels or world/chapter scenes
│   ├── player/                 # Player-related scenes and components
│   ├── props/                  # Reusable world objects and interactive items
│   ├── effects/                # Particles, glow effects, and transitions
│   └── ui/                     # Reusable UI scenes such as HUD or dialogue panels
├── scripts/                    # GDScript files grouped by responsibility
│   ├── autoload/               # Global singleton scripts such as managers
│   ├── core/                   # Shared helpers, constants, and utility logic
│   ├── gameplay/               # Mechanics, interaction logic, and progression systems
│   ├── systems/                # Save/load, dialogue, inventory, or other core systems
│   └── ui/                     # Menu behavior, navigation, and interface scripts
├── resources/                  # Reusable Godot resources shared across scenes
│   ├── data/                   # Structured gameplay data and configuration resources
│   ├── materials/              # Materials and shaders used in the game
│   └── themes/                 # UI themes, fonts, colors, and style resources
├── docs/                       # Supporting documentation and planning notes
├── tests/                      # Prototype scenes and testing files
├── addons/                     # External Godot plugins and editor extensions
├── project.godot               # Main Godot project configuration file
└── README.md                   # Documentation explaining the repository architecture
```

### `.godot/`

This folder is generated by Godot. It contains editor data, cache, and import metadata used by the engine while working on the project. It is not where the actual game content is designed, but it supports the development process.

### `assets/`

This folder stores the raw visual and audio files used by the game.

#### `assets/audio/`

Contains music, ambient sounds, button sounds, and gameplay sound effects.

#### `assets/backgrounds/`

Stores large background images, scenery, and parallax art used in menus or levels.

#### `assets/fonts/`

Contains fonts used by the interface, title screens, and in-game text.

#### `assets/icons/`

Stores small interface graphics such as menu icons, cursor icons, or status indicators.

#### `assets/sprites/`

Contains character sprites, props, decorative objects, and sprite sheets for animation.

#### `assets/tilesets/`

Stores tile atlases and environment tiles used to build levels.

#### `assets/ui/`

Contains interface art such as frames, panels, buttons, parchment banners, overlays, and other menu elements.

### `scenes/`

This folder contains the main Godot scenes that build the game.

#### `scenes/core/`

Contains base scenes that support the game flow, such as loaders, transitions, or boot scenes.

#### `scenes/menus/`

Stores menu-related scenes, such as the main menu, options menu, pause menu, and credits screen.

#### `scenes/levels/`

Contains the playable stages of the game. This can be organized by day, world, or chapter depending on the design of the project.

#### `scenes/player/`

Stores the player scene and any related scene components, such as hitboxes, animations, and interaction areas.

#### `scenes/props/`

Contains reusable world objects, decorations, interactive items, triggers, and environmental details.

#### `scenes/effects/`

Stores temporary visual scenes such as particles, glow effects, transitions, and animated overlays.

#### `scenes/ui/`

Contains reusable user interface scenes that may be shared across menus or gameplay, such as confirmation panels, dialogue boxes, and HUD elements.

### `scripts/`

This folder contains all GDScript files, grouped by responsibility instead of placing scripts randomly next to scenes.

#### `scripts/autoload/`

Contains global scripts registered as autoloads in Godot, such as managers for audio, game state, scene transitions, or save data.

#### `scripts/core/`

Stores helper scripts, utilities, shared constants, and general logic that can be reused across multiple systems.

#### `scripts/gameplay/`

Contains gameplay logic such as player mechanics, interactions, level rules, progression systems, or mission-related behavior.

#### `scripts/systems/`

Stores larger independent systems, such as save/load handling, dialogue logic, inventory systems, or configuration handling.

#### `scripts/ui/`

Contains scripts related to menu behavior, button interactions, hover effects, interface animations, and navigation logic.

### `resources/`

This folder is used for Godot resources that are shared between scenes.

#### `resources/data/`

Contains structured data files or reusable configuration resources for gameplay values, level data, or object definitions.

#### `resources/materials/`

Stores materials and shaders used by visual elements in the game.

#### `resources/themes/`

Contains reusable UI themes, font settings, colors, and style resources for keeping the interface consistent.

### `docs/`

Stores documentation related to the project, such as design notes, planning documents, references, and development guides.

### `tests/`

Can be used for prototype scenes, mechanic tests, or validation scenes created during development.

### `addons/`

Contains external Godot plugins or additional editor tools used by the project.

### `project.godot`

This is the main Godot project configuration file. It defines the project settings and is necessary for the engine to open the game correctly.
